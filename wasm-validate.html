WebAssembly Engine Stress Test

This tool attempts to reproduce a deadlock in JavaScript engines by continuously running WebAssembly.validate on the main thread, while simultaneously firing WebAssembly.compile across multiple Web Workers. Unique WebAssembly binaries are generated on the fly to bypass module caching.

Controls

More workers = higher concurrency pressure.
